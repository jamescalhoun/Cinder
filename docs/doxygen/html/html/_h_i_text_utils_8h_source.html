45 /*
46  * GetTextAndEncodingFromCFString()
47  *
48  * Summary:
49  * Converts the contents of a CFString to the one- or two-byte
50  * encoding that most accurately represents the original Unicode
51  * characters in the string.
52  *
53  * Discussion:
54  * Because the Macintosh toolbox has had, until Carbon, little
55  * support for Unicode, you may often find that your applications
56  * need to translate a CFString (containing Unicode characters) back
57  * to a one- or two-byte encoding of the string in order to draw,
58  * measure, or otherwise use the text in the string.
59  * GetTextAndEncodingFromCFString is a utility that may be helpful
60  * to do this, in some circumstances. The Carbon toolbox uses this
61  * API itself when translating CFStrings to Pascal strings.
62  * GetTextAndEncodingFromCFString attempts to convert the Unicode
63  * characters in the string to the closest non-Unicode
64  * representation of the string. For example, a string containing
65  * only Unicode characters that are present in the MacRoman one-byte
66  * encoding will be translated to MacRoman, and
67  * kTextEncodingMacRoman will be returned as the encoding of the
68  * text. A string containing Unicode characters that are present in
69  * the MacJapanese two-byte encoding will be translated to
70  * MacJapanese (Shift-JIS), and kTextEncodingMacJapanese will be
71  * returned as the encoding of the text.
72  * GetTextAndEncodingFromCFString is designed to be used for simple
73  * strings which contain only text from a single language. It is not
74  * designed to translate strings with multiple runs of text from
75  * different languages, and will only return the first run of such a
76  * string; for example, if you pass it a string containing a run of
77  * Roman characters followed by a run of Japanese characters, it
78  * will only return the Roman characters in the output buffer. For
79  * more complex text translation, you should call the Unicode
80  * converter directly (see ConvertFromUnicodeToTextRun, which
81  * GetTextAndEncodingFromCFString uses internally).
82  *
83  * Parameters:
84  *
85  * inString:
86  * The string to convert.
87  *
88  * outText:
89  * The output buffer in which to place the converted text.
90  *
91  * inTextMaxLength:
92  * The length in bytes of the input buffer.
93  *
94  * outTextLength:
95  * On exit, contains the length in bytes of the text that was
96  * placed into the output buffer.
97  *
98  * outEncoding:
99  * On exit, contains the encoding of the text that was placed into
100  * the output buffer.
101  *
102  * Availability:
103  * Non-Carbon CFM: not available
104  * CarbonLib: in CarbonLib 1.2.5 and later
105  * Mac OS X: in version 10.0 and later
106  */
107 EXTERN_API( OSStatus )
108 GetTextAndEncodingFromCFString(
109  CFStringRef inString,
110  BytePtr outText,
111  ByteCount inTextMaxLength,
112  ByteCount * outTextLength,
113  TextEncoding * outEncoding); /* can be NULL */
