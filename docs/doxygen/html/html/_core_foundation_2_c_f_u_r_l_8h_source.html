24 /* CFURLs are composed of two fundamental pieces - their string, and a */
25 /* (possibly NULL) base URL. A relative URL is one in which the string */
26 /* by itself does not fully specify the URL (for instance "myDir/image.tiff"); */
27 /* an absolute URL is one in which the string does fully specify the URL */
28 /* ("file://localhost/myDir/image.tiff"). Absolute URLs always have NULL */
29 /* base URLs; however, it is possible for a URL to have a NULL base, and still */
30 /* not be absolute. Such a URL has only a relative string, and cannot be */
31 /* resolved. Two CFURLs are considered equal if and only if their strings */
32 /* are equal and their bases are equal. In other words, */
33 /* "file://localhost/myDir/image.tiff" is NOT equal to the URL with relative */
34 /* string "myDir/image.tiff" and base URL "file://localhost/". Clients that */
35 /* need these less strict form of equality should convert all URLs to their */
36 /* absolute form via CFURLCopyAbsoluteURL(), then compare the absolute forms. */

113 /*
114 All URLs can be broken into two pieces - the scheme (preceding the
115 first colon) and the resource specifier (following the first colon).
116 Most URLs are also "standard" URLs conforming to RFC 1808 (available
117 from www.w3c.org). This category includes URLs of the file, http,
118 https, and ftp schemes, to name a few. Standard URLs start the
119 resource specifier with two slashes ("//"), and can be broken into
120 four distinct pieces - the scheme, the net location, the path, and
121 further resource specifiers (typically an optional parameter, query,
122 and/or fragment). The net location appears immediately following
123 the two slashes and goes up to the next slash; it's format is
124 scheme-specific, but is usually composed of some or all of a username,
125 password, host name, and port. The path is a series of path components
126 separated by slashes; if the net location is present, the path always
127 begins with a slash. Standard URLs can be relative to another URL,
128 in which case at least the scheme and possibly other pieces as well
129 come from the base URL (see RFC 1808 for precise details when resolving
130 a relative URL against its base). The full URL is therefore
131 
132 <scheme> "://" <net location> <path, always starting with slash> <add'l resource specifiers>
133 
134 If a given CFURL can be decomposed (that is, conforms to RFC 1808), you
135 can ask for each of the four basic pieces (scheme, net location, path,
136 and resource specifer) separately, as well as for its base URL. The
137 basic pieces are returned with any percent escape sequences still in
138 place (although note that the scheme may not legally include any
139 percent escapes); this is to allow the caller to distinguish between
140 percent sequences that may have syntactic meaning if replaced by the
141 character being escaped (for instance, a '/' in a path component).
142 Since only the individual schemes know which characters are
143 syntactically significant, CFURL cannot safely replace any percent
144 escape sequences. However, you can use
145 CFURLCreateStringByReplacingPercentEscapes() to create a new string with
146 the percent escapes removed; see below.
147 
148 If a given CFURL can not be decomposed, you can ask for its scheme and its
149 resource specifier; asking it for its net location or path will return NULL.
150 
151 To get more refined information about the components of a decomposable
152 CFURL, you may ask for more specific pieces of the URL, expressed with
153 the percent escapes removed. The available functions are CFURLCopyHostName(),
154 CFURLGetPortNumber() (returns an Int32), CFURLCopyUserName(),
155 CFURLCopyPassword(), CFURLCopyQuery(), CFURLCopyParameters(), and
156 CFURLCopyFragment(). Because the parameters, query, and fragment of an
157 URL may contain scheme-specific syntaxes, these methods take a second
158 argument, giving a list of characters which should NOT be replaced if
159 percent escaped. For instance, the ftp parameter syntax gives simple
160 key-value pairs as "<key>=<value>;" Clearly if a key or value includes
161 either '=' or ';', it must be escaped to avoid corrupting the meaning of
162 the parameters, so the caller may request the parameter string as
163 
164 CFStringRef myParams = CFURLCopyParameters(ftpURL, CFSTR("=;%"));
165 
166 requesting that all percent escape sequences be replaced by the represented
167 characters, except for escaped '=', '%' or ';' characters. Pass the empty
168 string (CFSTR("")) to request that all percent escapes be replaced, or NULL
169 to request that none be.
170 */
