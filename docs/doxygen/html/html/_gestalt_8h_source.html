310 /*
311  FORMAT OF gestaltATalkVersion RESPONSE
312  --------------------------------------
313  The version is stored in the high three bytes of the response value. Let us number
314  the bytes in the response value from 0 to 3, where 0 is the least-significant byte.
315 
316  Byte#: 3 2 1 0
317  Value: 0xMMNNRR00
318 
319  Byte 3 (MM) contains the major revision number, byte 2 (NN) contains the minor
320  revision number, and byte 1 (RR) contains a constant that represents the release
321  stage. Byte 0 always contains 0x00. The constants for the release stages are:
322 
323  development = 0x20
324  alpha = 0x40
325  beta = 0x60
326  final = 0x80
327  release = 0x80
328 
329  For example, if you call Gestalt with the 'atkv' selector when AppleTalk version 57
330  is loaded, you receive the long integer response value 0x39008000.
331 */

423 /*
424  The gestaltNativeCPUtype ('cput') selector can be used to determine the
425  native CPU type for all Macs running System 7.5 or later.
426 
427  The 'cput' selector is not available when running System 7.0 (or earlier)
428  on most 68K machines. If 'cput' is not available, then the 'proc' selector
429  should be used to determine the processor type.
430 
431  An application should always try the 'cput' selector first. This is because,
432  on PowerPC machines, the 'proc' selector will reflect the CPU type of the
433  emulator's "virtual processor" rather than the native CPU type.
434 
435  The values specified below are accurate. Prior versions of the Gestalt
436  interface file contained values that were off by one.
437 
438  The Quadra 840AV and the Quadra 660AV contain a bug in the ROM code that
439  causes the 'cput' selector to respond with the value 5. This behavior
440  occurs only when running System 7.1. System 7.5 fixes the bug by replacing
441  the faulty 'cput' selector function with the correct one.
442 
443  The gestaltNativeCPUfamily ('cpuf') selector can be used to determine the
444  general family the native CPU is in. This can be helpful for determing how
445  blitters and things should be written. In general, it is smarter to use this
446  selector (when available) than gestaltNativeCPUtype since newer processors
447  in the same family can be handled without revising your code.
448 
449  gestaltNativeCPUfamily uses the same results as gestaltNativeCPUtype, but
450  will only return certain CPU values.
451 */

842  MACHINE TYPE CONSTANTS NAMING CONVENTION

844  All future machine type constant names take the following form:
845

848  Line Names
849 
850  The following table contains the lines currently produced by Apple and the
851  lineName substrings associated with them:
852 
853  Line lineName

855  Macintosh LC "MacLC"

859  Power Macintosh "PowerMac"

863

866  Macintosh Quadra "MacQuadra" (preferred)
867  "Quadra" (also used, but not preferred)
868  Macintosh Centris "MacCentris"

871 
872  The modelNumber is a string representing the specific model of the machine
873  within its particular line. For example, for the Power Macintosh 8100/80,
874  the modelNumber is "8100".
875 
876  Some Performa & LC model numbers contain variations in the rightmost 1 or 2
877  digits to indicate different RAM and Hard Disk configurations. A single
878  machine type is assigned for all variations of a specific model number. In
879  this case, the modelNumber string consists of the constant leftmost part

882  constant: gestaltPerforma6100.

891  Power Macintosh 8100/100 return different machine type constants. This is
892  why some existing machine type constant names take the form:
893 
894  gestalt<lineName><modelNumber>_<speed>
895 
896  e.g.
897

905  Macintosh 7200/75 and a Power Macintosh 7200/90 return the same machine type
906  constant: gestaltPowerMac7200.

923  gestaltPowerBook180c
