353 /********************************************************************************
354 
355  String Types and Unicode Types
356 
357  UnicodeScalarValue, A complete Unicode character in UTF-32 format, with
358  UTF32Char values from 0 through 0x10FFFF (excluding the surrogate
359  range 0xD800-0xDFFF and certain disallowed values).
360 
361  UniChar, A 16-bit Unicode code value in the default UTF-16 format.
362  UTF16Char UnicodeScalarValues 0-0xFFFF are expressed in UTF-16
363  format using a single UTF16Char with the same value.
364  UnicodeScalarValues 0x10000-0x10FFFF are expressed in
365  UTF-16 format using a pair of UTF16Chars - one in the
366  high surrogate range (0xD800-0xDBFF) followed by one in
367  the low surrogate range (0xDC00-0xDFFF). All of the
368  characters defined in Unicode versions through 3.0 are
369  in the range 0-0xFFFF and can be expressed using a single
370  UTF16Char, thus the term "Unicode character" generally
371  refers to a UniChar = UTF16Char.
372 
373  UTF8Char An 8-bit code value in UTF-8 format. UnicodeScalarValues
374  0-0x7F are expressed in UTF-8 format using one UTF8Char
375  with the same value. UnicodeScalarValues above 0x7F are
376  expressed in UTF-8 format using 2-4 UTF8Chars, all with
377  values in the range 0x80-0xF4 (UnicodeScalarValues
378  0x100-0xFFFF use two or three UTF8Chars,
379  UnicodeScalarValues 0x10000-0x10FFFF use four UTF8Chars).
380 
381  UniCharCount A count of UTF-16 code values in an array or buffer.
382 
383  StrNNN Pascal string holding up to NNN bytes
384  StringPtr Pointer to a pascal string
385  StringHandle Pointer to a StringPtr
386  ConstStringPtr Pointer to a read-only pascal string
387  ConstStrNNNParam For function parameters only - means string is const
388 
389  CStringPtr Pointer to a C string (in C: char*)
390  ConstCStringPtr Pointer to a read-only C string (in C: const char*)
391 
392  Note: The length of a pascal string is stored as the first byte.
393  A pascal string does not have a termination byte.
394  A pascal string can hold at most 255 bytes of data.
395  The first character in a pascal string is offset one byte from the start of the string.
396 
397  A C string is terminated with a byte of value zero.
398  A C string has no length limitation.
399  The first character in a C string is the zeroth byte of the string.
400 
401 
402 *********************************************************************************/
