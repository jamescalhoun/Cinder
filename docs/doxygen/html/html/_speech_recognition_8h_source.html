321 /********************************************************************************/
322 /* NOTES ON USING THE API */
323 /* */
324 /* All operations (with the exception of SRGetRecognitionSystem) are */
325 /* directed toward an object allocated or begot from New, Get and Read */
326 /* type calls. */
327 /* */
328 /* There is a simple rule in dealing with allocation and disposal: */
329 /* */
330 /* * all toolbox allocations are obtained from a SRRecognitionSystem */
331 /* */
332 /* * if you obtain an object via New or Get, then you own a reference */
333 /* to that object and it must be released via SRReleaseObject when */
334 /* you no longer need it */
335 /* */
336 /* * when you receive a SRRecognitionResult object via AppleEvent or */
337 /* callback, it has essentially been created on your behalf and so */
338 /* you are responsible for releasing it as above */
339 /* */
340 /* * when you close a SRRecognitionSystem, all remaining objects which */
341 /* were allocated with it will be forcefully released and any */
342 /* remaining references to those objects will be invalid. */
343 /* */
344 /* This translates into a very simple guideline: */
345 /* If you allocate it or have it allocated for you, you must release */
346 /* it. If you are only peeking at it, then don't release it. */
347 /* */
