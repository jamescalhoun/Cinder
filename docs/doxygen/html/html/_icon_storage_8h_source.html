88 /*
89  IconFamily 'icns' resources contain an entire IconFamily (all sizes and depths).
90  For custom icons, icns IconFamily resources of the custom icon resource ID are fetched first before
91  the classic custom icons (individual 'ics#, ICN#, etc) are fetched. If the fetch of the icns resource
92  succeeds then the icns is looked at exclusively for the icon data.
93  For custom icons, new icon features such as 32-bit deep icons are only fetched from the icns resource.
94  This is to avoid incompatibilities with cut & paste of new style icons with an older version of the
95  MacOS Finder.
96  DriverGestalt is called with code kdgMediaIconSuite by IconServices after calling FSM to determine a
97  driver icon for a particular device. The result of the kdgMediaIconSuite call to DriverGestalt should
98  be a pointer an an IconFamily. In this manner driver vendors can provide rich, detailed drive icons
99  instead of the 1-bit variety previously supported.
100 */
101 enum {
102  kIconFamilyType = FOUR_CHAR_CODE('icns')
103 };
