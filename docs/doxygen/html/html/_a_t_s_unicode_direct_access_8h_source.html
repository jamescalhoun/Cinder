159 /*
160  * ATSUDirectGetLayoutDataArrayPtrFromLineRef()
161  *
162  * Summary:
163  * Returns the data pointer specified by iDataSelector and
164  * referenced by iLineRef.
165  *
166  * Discussion:
167  * This function simply returns the data pointer specified by
168  * iDataSelector and referenced by iLineRef. This data pointer
169  * should not be freed directly after it's been used. Rather, it
170  * should be released using ATSUDirectReleaseLayoutDataArrayPtr.
171  * Doing so serves as a signal to ATSUI that the caller is done with
172  * the data and that it can merge it in smoothly and adjust its
173  * internal processes. Furthermore, it may be the case that the
174  * pointer returned may be dynamically allocated one or contain
175  * dynamically allocated data. If it's not properly freed, a memory
176  * leak may result. This function may only be called within the
177  * context of an ATSUDirectLayoutOperationOverrideUPP callback. The
178  * pointer returned points to the exact data referenced by the
179  * ATSUTextLayout object that triggered the callback call. This is
180  * by far the most efficient way to use the direct access calls
181  * because for most requested data, no allocation and copy is done.
182  * Furthermore, because this a direct pointer to the data that ATSUI
183  * will use for it's layout, the data arrays returned by this can be
184  * tweaked and edited. Many of the requested arrays are created by
185  * ATSUI only when necessary. If these arrays are to be altered,
186  * then be sure to set iCreate to true. This will ensure that this
187  * array is created. If the arrays are not created, ATSUI
188  * automatically assumes that all entries in the array are zero. The
189  * pointer returned by this function is only valid within the
190  * context of the callback. Do not attempt to retain it for later
191  * use.
192  *
193  * Parameters:
194  *
195  * iLineRef:
196  * The ATSULineRef which was passed into a
197  * ATSUDirectLayoutOperationOverrideUPP callback function as a
198  * parameter.
199  *
200  * iDataSelector:
201  * The selector for the data that is being requested.
202  *
203  * iCreate:
204  * If the ATSULineRef passed in iLineRef does not reference the
205  * requested array, then a zero-filled one will be created and
206  * returned in oLayoutDataArray if this is set to true. For some
207  * ATSUDirectDataSelectors, these cannot be simply created. Thus,
208  * this flag will have no affect on these few
209  * ATSUDirectDataSelectors.
210  *
211  * oLayoutDataArrayPtr:
212  * Upon sucessful return, this parameter will contain a pointer to
213  * an array of the requested values if the ATSULineRef passed in
214  * iLineRef references those values. If this is not the case, then
215  * NULL will be returned, unless iCreate is set to true and the
216  * array can be created. This parameter itself may be set to NULL
217  * if only a count of the entries is needed.
218  *
219  * oLayoutDataCount:
220  * Upon sucessful return, this parameter will contain a count of
221  * the entries in the array returned in oLayoutDataArray.
222  *
223  * Availability:
224  * Non-Carbon CFM: not available
225  * CarbonLib: not available in CarbonLib 1.x, is available on Mac OS X version 10.2 and later
226  * Mac OS X: in version 10.2 and later
227  */
228 EXTERN_API_C( OSStatus )

231  ATSUDirectDataSelector iDataSelector,
232  Boolean iCreate,

234  ItemCount * oLayoutDataCount);

238 /*
239  * ATSUDirectGetLayoutDataArrayPtrFromTextLayout()
240  *
241  * Summary:
242  * Returns the data pointer specified by iDataSelector and
243  * referenced by iTextLayout for the line starting at iLineOffset.
244  *
245  * Discussion:
246  * This function simply returns the data pointer specified by
247  * iDataSelector and referenced by iTextLayout for the line starting
248  * at iLineOffset. This data pointer should not be freed directly
249  * after it's been used. Rather, it should be released using
250  * ATSUDirectReleaseLayoutDataArrayPtr. Doing so serves as a signal
251  * to ATSUI that the caller is done with the data. Furthermore, it
252  * may be the case that the pointer returned may be dynamically
253  * allocated one or contain dynamically allocated data. If it's not
254  * properly freed, a memory leak may result. This function may not
255  * be called inside the context of an
256  * ATSUDirectLayoutOperationOverrideUPP callback for the
257  * ATSUTextLayout data that triggered the callback. All data
258  * returned will be a copy of the data in the object requested. This
259  * means two things: first of all, this means that it's a very
260  * inefficient way of using the data. All of the selectors that
261  * would have returned in constant time now would be forced to
262  * return in order-n time. Second of all, this means that the
263  * developer cannot change any of the data. Any changes the
264  * developer makes to the arrays returned by this API will have no
265  * effect on the layout. Using the
266  * kATSULayoutOperationPostLayoutAdjustment operation selector
267  * override and the ATSUDirectGetLayoutDataArrayPtrFromLineRef is a
268  * great alternative to using this API. Many of the requested arrays
269  * are created by ATSUI only when necessary. This means that it's
270  * possible that this API will return NULL pointer and a count of 0.
271  * In this case, if there's no error returned, the array simply
272  * doesn't exist and the caller should treat all of the entries in
273  * the array that they would have recieved as being 0.
274  *
275  * Parameters:
276  *
277  * iTextLayout:
278  * The ATSUTextLayout object from which the requested data will
279  * come from.
280  *
281  * iLineOffset:
282  * The edge offset that corresponds to the beginning of the range
283  * of text of the line of the requested data. If the text has
284  * multiple lines, then ATSUDirectGetLayoutDataArrayPtrFromLineRef
285  * will need to be called for each of the lines in which the
286  * requested data is needed.
287  *
288  * iDataSelector:
289  * The selector for the data that is being requested.
290  *
291  * oLayoutDataArrayPtr:
292  * Upon sucessful return, this parameter will contain a pointer to
293  * an array of the requested values if the ATSUTextLayout passed
294  * in iTextLayout references those values for the line offset
295  * iLineOffset. If this is not the case, then NULL will be
296  * returned. This parameter itself may be set to NULL if only a
297  * count of the entries is needed.
298  *
299  * oLayoutDataCount:
300  * Upon sucessful return, this parameter will contain a count of
301  * the entries in the array returned in oLayoutDataArray.
302  *
303  * Availability:
304  * Non-Carbon CFM: not available
305  * CarbonLib: not available in CarbonLib 1.x, is available on Mac OS X version 10.2 and later
306  * Mac OS X: in version 10.2 and later
307  */

362  *
363  * Summary:

373  * outstanding ATSUStyleSettingRef array obtained by using the
374  * kATSUDirectDataStyleSettingATSUStyleSettingRefArray selector, the
375  * pointer obtained for this may no longer be valid after this
376  * function has been called. These pointers should be freed before
377  * calling this function and re-obtained afterwards.
378  *
379  * Parameters:
380  *
381  * iLineRef:
382  * An ATSULineRef which was passed into a
383  * ATSUDirectLayoutOperationOverrideUPP callback function as a
384  * parameter.
385  *
386  * iStyleSettingRef:
387  * The ATSUStyleSettingRef to be looked up or added to the
388  * ATSUTextLayout referenced by iTextLayout for the line starting
389  * at the offset iLineOffset.
390  *
391  * oStyleIndex:
392  * Upon sucessful return, this will parameter will be set to the
393  * index of the ATSUStyleSettingRef passed in iStyleSettingRef for
394  * the line referenced by iLineRef. If the ATSUStyleSettingRef
395  * does not exist, in that context, then it will be added and the
396  * new index will be returned here.
397  *
398  * Availability:
399  * Non-Carbon CFM: not available
400  * CarbonLib: not available in CarbonLib 1.x, is available on Mac OS X version 10.2 and later
401  * Mac OS X: in version 10.2 and later
402  */
