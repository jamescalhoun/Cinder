173  ***** Normalise 68K Calling C Conventions *****
174  Define special types that handle the difference in parameter passing
175  between different Mac OS C compilers when generating 68K code. OT
176  exports C calling conventions routines, and various C compilers use
177  various different conventions. Differences in the placement of the result
178  are covered above, where we output pragma pointers_in_D0. The other big
179  difference is how the compilers pass integer parameters less than 32 bits.

183  sub 32 bit values to routines. They are always defined to a 32 bit

186  languages (eg Pascal) OTBooleanParam is not compatible with Boolean.
