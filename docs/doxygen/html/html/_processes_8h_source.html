286 /*
287  * [Mac]GetCurrentProcess()
288  *
289  * Discussion:
290  * Return the canonical process serial number to the caller.
291  *
292  * All applications ( things which can appear in the Dock or which
293  * are not documents and are launched by the Finder or Dock ) on Mac
294  * OS 10 have a unique process serial number. This number is
295  * created when the application launches, and remains until the
296  * application quits. Other system services, like AppleEvents, use
297  * the ProcessSerialNumber to specify an application.
298  *
299  * During launch, every application 'checks in' with the Process
300  * Manager. Before this checkin, the application can not receive
301  * events or draw to the screen. Prior to Mac OS 10.2, this 'check
302  * in' happened before the applications's main() function was
303  * entered. In Mac OS 10.2 and later, this 'check in' does not
304  * happen until the first time the application calls a Process
305  * Manager function, or until it enters CFRunLoopRun() for the main
306  * runloop. This allows tools and other executables which do not
307  * need to receive events to link against more of the higher level
308  * toolbox frameworks, but may cause a problem if the application
309  * expects to be able to retrieve events or use CoreGraphics
310  * services before this checkin has occurred.
311  *
312  * An application can force the connection to the Process Manager to
313  * be set up by calling any Process Manager routine, but the
314  * recommended way to do this is to call GetCurrentProcess() to ask
315  * for the current application's PSN. This will initialize the
316  * connection to the Process Manager if it has not already been set
317  * up and 'check in' the application with the system.
318  *
319  * This function is named MacGetCurrentProcess() on non Macintosh
320  * platforms and GetCurrentProcess on the Macintosh. However, even
321  * Macintosh code can use the MacGetCurrentProcess() name since
322  * there is a macro which maps back to GetCurrentProcess().
323  *
324  * Lastly, it is usually not necessary to call GetCurrentProcess()
325  * to get the 'current' process psn merely to pass it to another
326  * Process Manager routine. Instead, just construct a
327  * ProcessSerialNumber with 0 in highLongOfPSN and kCurrentProcess
328  * in lowLongOfPSN and pass that. For example, to make the current
329  * process the frontmost process, use ( C code follows )
330  *
331  * ProcessSerialNumber psn = { 0, kCurrentProcess };
332  *
333  * OSErr err = SetFrontProcess( & psn );
334  *
335  * If you need to pass a ProcessSerialNumber to another application
336  * or use it in an AppleEvent, you do need to get the canonical PSN
337  * with this routine.
338  *
339  * Parameters:
340  *
341  * PSN:
342  * Pass in where the current application process serial number
343  * should be returned.
344  *
345  * Result:
346  * An operating system status code.
347  *
348  * Availability:
349  * Non-Carbon CFM: in InterfaceLib 7.1 and later
350  * CarbonLib: in CarbonLib 1.0 and later
351  * Mac OS X: in version 10.0 and later
352  */
353 #if TARGET_OS_MAC
354  #define MacGetCurrentProcess GetCurrentProcess
355 #endif
356 EXTERN_API( OSErr )
357 MacGetCurrentProcess(ProcessSerialNumber * PSN) THREEWORDINLINE(0x3F3C, 0x0037, 0xA88F);

384 /*
385  * GetProcessInformation()
386  *
387  * Discussion:
388  * Fill in the provided record with information about the process
389  * with the provided process serial number.
390  *
391  * The caller must fill in the .processInfoLength field with the
392  * value sizeof ( ProcessInformationRecord ) before making this
393  * call. Also, the .processName field must point to either NULL or
394  * to a Str31 structure in the caller's memory space, and the
395  * .processAppSpec field must point to a FSSpec in the caller's
396  * memory space.
397  *
398  * If the caller does not care about the process name or the process
399  * application spec values, then setting those fields in the
400  * structure to NULL before this call means less work must be done
401  * to construct these values and so the call is more
402  * efficient.
403  *
404  * The processName field may not be what you expect, especially if
405  * an application has a localized name. The .processName field, if
406  * not NULL, on return will contain the filename part of the
407  * executable file of the application. If you want the localized,
408  * user-displayable name for an application, call
409  * CopyProcessName().
410  *
411  * On Mac OS X, the processSize and processFreeMem fields are
412  * returned with the value 0.
413  *
414  * Parameters:
415  *
416  * PSN:
417  * Pass in the process serial number of the process to return
418  * information for.
419  *
420  * info:
421  * Pass in a structure where the information will be returned.
422  *
423  * Availability:
424  * Non-Carbon CFM: in InterfaceLib 7.1 and later
425  * CarbonLib: in CarbonLib 1.0 and later
426  * Mac OS X: in version 10.0 and later
427  */
